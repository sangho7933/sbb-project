# Aion2 Community & Equipment Simulator

👉 사용자 상태 기반 추천 시스템 + 인증 기반 UI 분기 + 관리자 모드 + 데이터 시각화 + AI 인터랙션 + 세계관 UI를 결합한 웹 서비스

Spring Boot 기반으로 구현한 아이온2 커뮤니티 및 장비 시뮬레이터 웹 서비스입니다.

---

## 🔥 주요 기능

### ✅ 커뮤니티
- 게시글 CRUD (작성 / 수정 / 삭제 / 조회)
- 댓글 기능 (추천 / 비추천 (토글식)/ 수정 / 삭제)
- 페이징 처리
- 로그인 사용자만 게시글 및 댓글 작성 가능

### ✅ 회원 시스템
- Spring Security 기반 로그인 / 회원가입
- 인증 상태에 따른 UI 분기 처리 (로그인 시 기능 활성화)

### ✅ 관리자 기능
- 관리자 전용 "관리자 모드" 진입 구조 구현
- 관리자 모드에서 게시글 삭제 / 댓글 삭제 / 거래 관리 가능
- 관리자 모드 해제 기능을 통해 일반 사용자 모드로 전환 가능
- 관리자 페이지에서 데이터 그래프 시각화 제공

### ✅ 거래 시스템
- 아이템 등록 / 구매 기능
- 거래 내역 조회
- 사용자 기준 구매/판매 흐름 연결

### ✅ 마이페이지
- 보유 장비 조회
- 장비 장착 기능
- 구매 아이템과 장비 연동

### ✅ 장비 시뮬레이터
- 장비 스탯 기반 전투력 계산
- 전체 장비 중 가장 약한 부위 분석
- 해당 부위 기준 업그레이드 추천
- 단순 최고 수치가 아닌 “현재 대비 점진적 성장(+500 기준)” 추천 로직 구현

### ✅ AI 기반 기능 ⭐
- 사용자 입력 키워드 기반 페이지 이동 기능 구현  
  ("자유", "거래", "뉴비", "마도성" 등 입력 시 해당 페이지로 이동)

- 자연어 입력을 해석하여 게시글을 자동 생성하는 기능 구현  
  (예: "자유게시판에 글 써줘 ..." 입력 시 자동으로 게시글 생성)

- 사용자 입력 → 의도 분석 → 기능 수행까지 이어지는 AI 기반 인터랙션 시스템 구축

## 🤖 AI 가상 전투 시뮬레이션

- 현재 장비 상태를 기반으로 AI 상대 자동 생성
- 전투 연출을 통해 전투 상황을 직관적으로 표현
- 전투 결과 분석을 통해 부족한 장비 및 성장 방향 제시

### ✅ UI / UX
- 마이페이지 장비 UI 개선 및 가독성 향상
- 이미지 기반 직관적 UI 구성
- 다크 모드를 게임 세계관(천사 / 악마) 기반으로 설계하여 몰입도 향상

## ⚙️ 기술 스택

- Backend: Spring Boot, Spring Security
- Frontend: Thymeleaf, HTML, CSS, JavaScript
- Database: MySQL
- Build Tool: Gradle
- IDE: IntelliJ IDEA
- Java: JDK 21 (권장: 21.0.10+7)
  
---
## 🚀 실행 방법

1. Java 21 설치
2. MySQL 설치
3. MySQL에서 아래 명령어 실행

   CREATE DATABASE sbb;

4. src/main/resources/application.properties 파일에서 DB 정보 설정
5. 프로젝트 클론
6. Gradle build 또는 IntelliJ에서 Spring Boot 실행
7. localhost:8080 접속

## 💡 프로젝트 특징

- 단순히 최고 성능 장비를 추천하는 방식이 아니라,  
  현재 사용자의 장비 상태를 분석하여 가장 약한 부위를 우선적으로 탐색하고  
  해당 부위 내에서 점진적 성장이 가능한 장비를 추천하는 로직을 구현하였습니다.

- 거래 시스템, 장비 관리, 시뮬레이터 기능을 서로 연동하여  
  데이터 흐름이 자연스럽게 이어지도록 설계하였습니다.

- 추천 결과를 단순 데이터 출력이 아닌 사용자 UI/UX 흐름까지 연결하여  
  실제 서비스 형태로 동작하도록 구현하였습니다.

- 로그인 여부에 따라 마이페이지, 거래게시판, 시뮬레이션 기능 접근을 제어하고  
  게시글 및 댓글 작성 권한을 제한하는 인증 기반 구조를 구현하였습니다.

- 관리자 계정도 기본적으로 일반 사용자 UI를 유지하도록 설계하고,  
  별도의 "관리자 모드" 버튼을 통해 관리자 기능을 활성화하도록 구현하였습니다.

- 관리자 모드에서는 게시글 삭제, 댓글 삭제, 거래 관리 기능을 수행할 수 있도록  
  권한 기반 기능 제어 구조를 적용하였습니다.

- 또한 관리자 모드를 해제하여 일반 사용자 모드로 전환할 수 있도록 구현하여  
  사용자 상태에 따른 UI 및 기능 전환 구조를 설계하였습니다.

- 관리자 페이지에서 데이터를 그래프 형태로 시각화하여  
  주요 정보를 한눈에 확인할 수 있도록 구현하였습니다.

- 다크 모드를 단순 테마 전환이 아닌, 게임 세계관(천사 / 악마)을 반영한 UI로 구성하여  
  사용자 경험과 시각적 몰입도를 높였습니다.

- 사용자 입력을 기반으로 페이지 이동 및 게시글 자동 생성을 수행하는  
  AI 기반 인터랙션 시스템을 구현하였습니다.

## 💡 배운 점

- 게임 구조상 아이템 획득 후 거래가 이루어지는 흐름을 구현하고자 하였으나,  
  프로젝트 범위 및 데이터 구조 제약으로 인해 모든 아이템 거래가 가능하도록 설계하였습니다.  
  이 과정에서 실제 서비스 환경에서는 데이터 흐름 설계가 더욱 중요하다는 점을 체감하였습니다.

- AWS 서버 환경에서 프로젝트를 배포하는 과정에서 다양한 오류를 경험하며,  
  로컬 환경과 실제 서버 환경의 차이를 이해하게 되었습니다.

- 거래 게시판, 마이페이지, 시뮬레이션 기능 간 데이터(DB) 및 이미지 연결 과정에서  
  데이터 흐름 설계와 연동 구조의 중요성을 체감하였습니다.

- 사용자 자연어 입력을 기반으로 기능을 수행하는 AI 시스템 구현 과정에서  
  입력 의도 해석 및 처리 로직 설계의 어려움을 경험하였습니다.

- 장비 추천 기능 구현 시 단순 최고 성능이 아닌  
  사용자 상태 기반의 점진적 성장 로직을 설계하는 과정에서  
  알고리즘 설계와 기준 설정의 중요성을 배웠습니다.
